# 安全政策

## 报告安全漏洞

我们非常重视 OpenClaw GUI 项目的安全性。如果你发现了任何安全漏洞，请按照以下步骤报告：

### 如何报告

1. **不要**在公共 GitHub Issues 中报告安全漏洞
2. 请通过以下方式之一报告：
   - 发送邮件至：[security@openclaw.ai](mailto:security@openclaw.ai)
   - 在 GitHub 上通过私人消息联系维护者

### 报告时请包含

- 漏洞的描述
- 复现步骤
- 影响范围
- 可能的修复方案（如果有）
- 你的联系方式（可选）

## 安全最佳实践

### 使用 OpenClaw GUI 时

1. **保持更新**：始终使用最新版本的 OpenClaw GUI
2. **保护令牌**：妥善保管你的 OpenClaw 认证令牌
3. **限制访问**：不要在公共网络上暴露你的 OpenClaw GUI 实例
4. **使用 HTTPS**：在生产环境中始终使用 HTTPS
5. **定期备份**：定期备份你的配置和数据

### 开发时

1. **代码审查**：所有代码变更都需要经过审查
2. **依赖管理**：定期更新依赖项以修复安全漏洞
3. **秘密扫描**：不要将 API 密钥、密码等敏感信息提交到代码仓库
4. **输入验证**：始终验证和清理用户输入
5. **最小权限**：遵循最小权限原则

## 我们的承诺

- 我们会尽快确认收到你的报告
- 我们会在修复漏洞后告知你
- 我们会在安全公告中感谢你的贡献（如果你同意）

## 安全公告

我们会在以下位置发布安全公告：
- GitHub Releases
- 项目文档
- 社区 Discord
